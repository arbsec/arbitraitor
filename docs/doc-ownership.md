# Documentation Ownership and Conventions

This document defines who owns each documentation surface, which files are
generated vs handwritten, stability tiers, and diagram conventions. All
contributors — human and automated — must follow these rules.

## Doc surface ownership map

Every documentation surface has a single canonical owner. Changes to a
surface must update the owner file in the same PR.

| Surface | Location | Owner | Format |
|---------|----------|-------|--------|
| README | `README.md` | This file (handwritten) | Markdown |
| Contributor guide | `CONTRIBUTING.md` | This file (handwritten) | Markdown |
| Agent guide | `AGENTS.md` | This file (handwritten) | Markdown |
| Development conventions | `docs/conventions.md` | This file (handwritten) | Markdown |
| ADRs | `docs/adr/*.md` | **Canonical source** — copied to book at build time | Markdown |
| Published book | `book/src/*.md` | Handwritten (except ADRs and generated pages) | mdBook |
| Book navigation | `book/src/SUMMARY.md` | Handwritten — must list all pages | mdBook |
| Rust API docs | `rustdoc` comments in `crates/*/src/*.rs` | Source code (`#![warn(missing_docs)]`) | rustdoc |
| CLI help | `clap` doc comments in `crates/arbitraitor-cli/src/main.rs` | Source code | Generated by clap |
| Changelog | `CHANGELOG.md` | This file (handwritten) | Keep a Changelog |
| Security policy | `SECURITY.md` | This file (handwritten) | Markdown |
| CI workflows | `.github/workflows/*.yml` | This file (handwritten) | YAML |

### ADR canonical source

`docs/adr/` is the **single canonical source** for all ADRs. The
`docs.yml` workflow copies `docs/adr/*` to `book/src/adr/` at build time.
The `book/src/adr/` directory is gitignored and must not be hand-edited.

## Generated vs handwritten files

| File | Type | How to update |
|------|------|---------------|
| `book/src/adr/*.md` | **Generated** (copied from `docs/adr/`) | Edit `docs/adr/`, not book copies |
| `book/src/SUMMARY.md` | **Handwritten** | Edit directly; must list all book pages |
| `book/src/cli-reference.md` | **Handwritten** (future: generated by `xtask`) | Edit directly until xtask exists |
| All other `book/src/*.md` | **Handwritten** | Edit directly |
| `docs/adr/*.md` | **Handwritten** | Edit directly (canonical ADR source) |

Generated files must not be hand-edited. A future `cargo xtask docs` will
generate CLI reference, exit-code tables, and ADR indexes from source.

## Stability tiers

Every reference page in the book should carry a stability indicator:

| Tier | Meaning | Example surfaces |
|------|---------|------------------|
| **Frozen** | Will not change without a major version bump | Security invariants, ADRs |
| **Unstable** | May change between minor versions; verified against a specific commit | CLI flags, config schema, receipt schema, policy syntax |
| **Experimental** | Likely to change; no stability guarantee | Plugin protocol, MCP tools, daemon API |

Unstable pages should include:

```markdown
> **Stability: Unstable.** Verified against commit `<sha>`.
> Flags, fields, and formats may change before 1.0.
```

## Diagram conventions

### When to use mermaid vs text

| Diagram type | Tool | Reason |
|--------------|------|--------|
| Component graphs, data flow, state machines | **Mermaid** | Relational structure; auto-layout |
| Sequence/protocol flows | **Mermaid** | Temporal ordering |
| Crate/workspace trees | **Text** (`├──`) | Directory-listing format; scannable |
| Simple linear pipelines | **Text** | Low ceremony; renders everywhere |
| Tabular data | **Markdown tables** | Never use mermaid for data |

### Mermaid configuration

Mermaid is configured in `book/book.toml` (`[preprocessor.mermaid]`) with
`mermaid.min.js` and `mermaid-init.js`. The `docs.yml` workflow installs
`mdbook-mermaid`. No additional setup is needed.

GitHub renders mermaid blocks natively in markdown since February 2022.

### Migration policy

When touching a file that uses ASCII box-art for a genuinely diagrammatic
purpose (not a tree or table), convert it to mermaid in the same PR. Do not
do a standalone big-bang conversion — it creates churn with no behavior change.
