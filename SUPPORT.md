# Support

## Getting Help

- **Questions and discussion:** [GitHub Discussions](https://github.com/arbsec/arbitraitor/discussions)
- **Bug reports:** [GitHub Issues](https://github.com/arbsec/arbitraitor/issues) (use the bug report template)
- **Security vulnerabilities:** See [SECURITY.md](SECURITY.md) — **do not use public issues for security reports**

## Before Filing an Issue

1. Search existing issues and discussions to avoid duplicates.
2. Check if the issue exists on the latest `main` branch.
3. Include relevant details: OS, Arbitraitor version, command that triggered the issue, and receipt or error output (redacted of secrets).

## False Positives / False Negatives

Use the "False Positive or False Negative" issue template. Include the artifact hash (not the artifact itself unless it is safe to share), the finding ID, and the receipt.
